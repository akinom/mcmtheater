What to do with the "Locate tools.jar" dialog that DrJava pops up when I start it ?

DrJava wants to know where tools.jar is located. tools.jar contains the tools that DrJava needs to compile and execute programs. You must tell DrJava where to find it, so click "Yes".

tools.jar is usually located in C:\Program Files\Java\jdk....\lib\tools.jar. So navigate to that file, select it, and click "Open".

Before you do anything you must ...

  • Open your current project

    All programming files, that I give to you, are part of DrJava projects. Eeach time, that you start DrJava, your first action must be to open the project file that comes with your current program. Project information is stored in files that have a ".pjt" extension. All starter projects are stored in mcmUser\drJava\<SomeName>.pjt.

    To open a project use the "Open" option from the "Project" menu, see DrJava Window

    Do not use the "Open" short cut button, neither the "Open" option in the File menu. These add a single file to the current project instead of opening a complete project with all its file.

  • Compile everything

    Once you successfully opened a project compile everything. You can do that in any of the following three ways:

    • Choosing the "Compile All Documents" option from the "Tools" menu, see DrJava Window
    • Clicking the "Compile All" short cut button.
    • Right clicking on the project name in the tree view tab on the left, see DrJava Window

  • Test whether you can start your Actor

    All projects, that I give to you, contain a file called TryStart.hist. All TryMeStart.hist files start an Actor in its own window/Stage. First click on the Interactions Tab at the bottom, right click inside the Interactions tab,and choosee "Execute Interactions History", choose your TryMeStart.hist file, see DrJava Window

How do I get rid of '//End of Interaction//' in Java files ?

This can be done with the help of "Find and Replace" from the "Edit" menu, see DrJava Window.

"Find Replace" adds a Tab at the bottom of DrJava's Window, see DrJava Window.

Enter '//End of Interaction//' in the 'Find Next" text area on the left and make sure the 'Replace With' area on the right is empty, see DrJava Window.

Click "Find Next". This find the first occurence of '//End of Interaction//' in the file, that is currently displayed in the edit pane. The Click "Replace/Find Next". This replaces '//End of Interaction//' with nothing and finds the next occurence. Repeat "Replace/Find Next" until all have been deleted.

How do I replace all '//End of Interaction//' with ';' ?

This can be done with the help of "Find and Replace" from the "Edit" menu, see DrJava Window.

"Find Replace" adds a Tab at the bottom of DrJava's Window, see DrJava Window.

Enter '//End of Interaction//' in the 'Find Next" text area on the left and enter ';' in the 'Replace With' area on the right. see DrJava Window.

Click "Find Next". This find is the first occurence of '//End of Interaction//' in the file, that is currently displayed in the edit pane. The Click "Replace/Find Next". This replaces '//End of Interaction//' with a ';' and finds the next occurence. Repeat "Replace/Find Next" have been replpaced by semicolons.

What is that "Must Save All Files To Continue" Dialog ?

DrJava must make sure that the compiler 'sees' all changes you made to your files. This dialog appears if you have not saved all your files. This meaans that some editing changes have not been written to the file system and the compiler can therefore not 'see' them. Definetely click "Yes". It is also OK to choose "Always save before compiling". If you do you will not see this dialog again.

I edited the Java file but my Actor doesn't seem to do the new statements.

You probably did not compile the edited code. Compile everything by doing one of the following:

  • Choosing the "Compile All Documents" option from the "Tools" menu, see DrJava Window
  • Clicking the "Compile All" short cut button.
  • Right clicking on the project name in the tree view tab on the left, see DrJava Window

If the compilation prints "Last compilation completed successfully." to the Compiler Ouput Tab (see DrJava Window) you are fine and should now be able to execute the new statements.

If you do see error messages in the Compiler Ouput Tab, (see DrJava Window) you need to fix all of them and compile again.
