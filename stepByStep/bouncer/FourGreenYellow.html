The applet below starts four yellow and four green Bouncers. The makeGreenYellow method contains the same statements as the act method in the previous example. In fact this initial actor came about by copying its code, renaming the act method to makeGreenYellow, editing it slightly, and writing the act method shown below.

The act method contains a for loop that wraps one statement, a call to makeGreenYellow. Each time makeGreenYellow is called, its statements are executed which create a green and a yellow bouncing balloon. Since it is called multiple times makeGreenYellow must avoid creating ballons that start at the same position and head into the same direction which is the reason for changes in the way the bouncers are initialized. In this example they are forced to jump to a random position on stage instead of the stage's center before they start. The initial actor sets their direction angle randomly as opposed to using 15 or 30 degree angles.

For loops in general serve to execute something multiple times. The for loop in this example states that the makeGreenYellow method call must be repeated 4 times. The variable i is set to 1 when the loop starts, and its value is increased by one for each repetition. The loop finishes once i becomes bigger than 4. Thus the loop executes 4 times, once with the variable i set to 1, once with i set to 2, once with i set to 3, and a last time with i set to 4. In this simple loop i serves as a counter only whose value is used to make sure that makeGreenYellow is in fact called exactly 4 times.
