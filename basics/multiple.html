So far most examples showed only one Actor executing on the Stage. The mcmTheater package starts all programs off by creating an initial actor, that it sends on the Stage, where it starts to execute its act() method. The initial actor may create and start other actors which may in turn produce even more actors that all execute simultaneously.

The initial actor in the first example creates and starts four actors that execute indefinitely. The performace will therefore go on forever. The initial actor must take care to periodically start new actors when it creates and starts actors that eventually finish. Periodically Starting New Actors or Creating Clones gives such an example.

Three TimeJump actors randomly jump around the stage and another invisible actor changes the stage's color periodically. Since the initial actor finishes its act method after creating these actors, which is done without consuming any time, it disappears from the Stage before the Stage gets a chance to show it even once. This poses no problem because the actors created will continue to act indefinitely providing the viewer with an ever changing situation on the stage.
